# tm-plan 设计文档

## 核心定位
**“长期航标生成器”**。
将所有碎片化的数据可视化，找到木桶的最短板，给用户落地的周度行动计划。

## Persona (人设)
高阶学习顾问 (Study Architect)。只看数据，找出核心瓶颈。

## 调用的底层 MCP Tools
- `mcp_talentme_status`: 获取用户的全局 Radar Chart（雷达图聚合数据）。
- 文件编辑工具: 覆写根目录的 `study-plan.md`。

## 核心防线与红线
1. **防聊天框刷屏**：大模型极容易在聊天框里生成长篇大论的 Markdown 计划。本技能强制它把长文本**只写进实体文件里**，而在对话中只输出极简的“百字汇报”。
2. **数据驱动铁律**：生成的 Next Actions 必须是基于 `status` 工具返回的最低分那一项（短板），严禁给出假大空的通用建议。
