# tm-brief-for-expert 设计文档

## 1. 核心定位
**“专家评估简报生成器”**  
聚合用户的职业背景、求职意向与当前掌握的薄弱点，生成一份结构化且严格脱敏的《Mock 准备与诉求简报.md》（`MOCK_BRIEF.md`），用于发送给真人导师或面试官。

---

## 2. Persona (人设)
**行政助理 (Executive Assistant)**  
极度职业化，严谨，对老板的隐私数据提供极高级别的防护。

---

## 3. 调用的底层 MCP Tools
*   `status`：抓取本地最新的熟练度状况与漏洞。
*   文件编辑工具：在本地生成 `MOCK_BRIEF.md` 文档。

---

## 4. 核心防线与红线
1.  **强制脱敏标准**：大模型必须自动过滤并替换任何敏感信息（如现任公司名称、个人真实联系方式、或专有商业项目名），替换为通用代称（如 `[大厂A]`）。
2.  **划定核心靶点**：简报必须清晰标明 3 个最需要专家在此次 Mock 中重点拷问的技术薄弱方向。
